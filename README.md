# Game Development Portfolio — AI Dataset Assets

**Katherine Feemster** · Game Development Specialist

Cross-engine, code-first game development for an AI research program: projects
that open in **Godot 4**, **Defold**, **Solar 2D**, **Panda 3D** and **Stride
(Xenko)** are all generated from a single Python pipeline. Every sprite,
scene, collection and `.egg` voxel is deterministic and diffable in git.

## Reproducibility

```bash
python3 scripts/python/run_all.py
```

`run_all.py` orchestrates six stages:

1. `build_godot_project.py` — writes `project.godot`, `main.tscn`,
   `VoxelScene.gd`, `AssetExporter.gd`, `creatures.png`, `manifest.csv`.
2. `build_defold_project.py` — writes `game.project`, `main.collection`,
   `hero.script`, `creatures.atlas`, `behavior_capture.lua`.
3. `build_solar2d_project.py` — writes `main.lua`, `config.lua`,
   `build.settings`, `scene.lua`, `particles.lua`, `capture.lua`, three PNGs.
4. `build_panda3d_project.py` — writes `main.py`, `Config.prc`, `scene.egg`
   (178 voxels), `headless_capture.py`, `scene_graph.json`.
5. `build_stride_project.py` — writes `.sln`, `.csproj`, `.sdpkg`, a YAML
   `.sdscene` with \(12 \times 12\) voxel entities, `DatasetSandboxGame.cs` and
   `DataCaptureComponent.cs`.
6. `render_posters.py` — five posters for the docs site.

Dependencies are narrow on purpose: `numpy`, `Pillow`, `matplotlib`, `pyyaml`.
The engines themselves are only needed to **run** the generated projects — not
to regenerate them. CI stays fast.
